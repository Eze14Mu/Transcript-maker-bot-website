We respect your privacy rights and are fully committed to treating data collected by the Bot with strict transparency. This Privacy Policy outlines exactly how the Bot handles, processes, and retains data when integrated into a Discord server environment.

1. Scope of Data Processing

To successfully compile and format clean text transcripts, the Bot must dynamically read, parse, and process content flowing through designated Discord channels where it has been granted explicit read permissions. This specific data includes:

  • Message Contents: Raw text strings, formatting tags, precise timestamps, and unique Discord Message IDs.
  • User Identifiers: Public Discord display names, account usernames, and immutable unique Discord User IDs (Snowflakes).
  • Server Meta-Data: Discord Guild (Server) names, structural Channel names, and localized Channel IDs.
  • Media Attachments: Hosted URLs and source links for images, videos, or raw files transmitted dynamically within the tracked channel.

2. Permitted Core Data Use

Data parsed by the Bot is strictly funneled into compiling the text documents requested explicitly by authorized server administrators. We enforce a zero-monetization data policy:

  • No Commercial Distribution: We do not sell, rent, trade, lease, or distribute chat content, user metadata, or server behaviors to any third-party marketing firms or data brokers.
  • Platform Protection: Collected text assets are meant solely to assist server personnel in maintaining order and reporting severe platform violations directly to Discord. We do not independently inspect or report logs unless legally mandated.

3. Data Retention and Storage Infrastructure

Secure Retention Protocol: Transcripts and their processed elements are securely stored on our servers for a rolling period of exactly 30 days. This gives server administrators time to download files for verification. Following this 30-day period, the data is permanently and irreversibly purged from our storage arrays.

4. User Privacy Rights and Opt-Outs

Because the Bot operates directly under the structural command of independent server owners, individual users who prefer not to have their public chat interactions transcribed must contact the relevant server administration team or abstain from communicating within channels where the Bot holds operational visibility. Server administrators retain the right to clear existing active transcripts or remove the Bot completely at any point to completely halt processing.
